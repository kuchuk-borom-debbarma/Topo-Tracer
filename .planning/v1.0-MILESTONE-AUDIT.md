## Integration Check

The inline integration check found no implementation blocker in the Hono read
model pipeline:

- Edge ingestion accepts and persists explicit `fromNodeId`/`toNodeId` endpoint
  data.
- ClickHouse schema includes raw edge endpoints, read nodes, read edges, trace
  summaries, and materialization checkpoints.
- `TraceReadModelMaterializer` loads checkpoints, reads raw events after the
  checkpoint, merges latest state, writes read rows, and saves checkpoints
  last.
- Bounded projection repository methods use repo-level caps and avoid
  unbounded full-trace projection reads.
- `LogServiceImpl.projectTraceGraph` orchestrates bounded node/edge reads and
  delegates graph shaping to `LogGraphProjector`.
- `LogGraphProjector` implements threshold visibility, ghost ranges, edge
  snapping, duplicate edge aggregation, orphan omission, and CAP metadata.
- Source assertions prevent route, frontend, SDK, `carno.js`, ancestry-path,
  stored-ghost, and pagination/windowing drift in Hono log source.
