# Plan: Phase 2 - Advanced SDK Features

Phase 2 focuses on performance and reliability enhancements for the Node.js SDK, along with support for distributed tracing context propagation.

## Phase 2.1: Batching & Buffer Management
- [ ] **2.1.1: Implement EventBuffer**
  - Create internal storage for events in `Tracer.ts`.
  - Update `TracerConfig` to include `batchSize` and `flushInterval`.
- [ ] **2.1.2: Implement Periodic & Size-based Flush**
  - Add `flush()` method to `Tracer`.
  - Set up `setInterval` for automatic flushing.
  - Trigger flush when `batchSize` is reached.
- [ ] **2.1.3: Enforce Hard Caps**
  - Ensure batches never exceed 1000 events.
  - Implement overflow dropping logic.

## Phase 2.2: Retry Logic & Reliability
- [ ] **2.2.1: Implement Exponential Backoff with Jitter**
  - Create a helper for calculating retry delays.
  - Wrap `fetch` calls in a retry loop.
- [ ] **2.2.2: Implement Failure Hook**
  - Update `TracerConfig` to include `onDrop` callback.
  - Call `onDrop` when a batch fails max retries.

## Phase 2.3: Distributed Tracing & Lifecycle
- [ ] **2.3.1: Support External Parent Context**
  - Update `startNode` to accept `parentSpanId`.
  - Create edges from `parentSpanId` when provided.
- [ ] **2.3.2: Context Injection/Extraction**
  - Implement `tracer.extractContext()` and `tracer.injectContext()`.
- [ ] **2.3.3: Lifecycle Hooks**
  - Implement `tracer.shutdown()`.
  - Register `SIGTERM`, `SIGINT`, and `beforeExit` listeners.

## Verification Plan
- **Unit Tests:**
  - Verify `EventBuffer` size and interval triggers.
  - Test retry delay calculation.
  - Test context injection/extraction.
- **Integration Tests:**
  - Verify that events from a remote "mock" service (via injected context) correctly link to local spans.
  - Verify that `shutdown()` flushes all pending events.
