## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service-to-repository | Projection orchestration asks for bounded latest-node input by user and trace. |
| Repository-to-ClickHouse | Projection-node query reads capped latest rows from ClickHouse. |
| Contract-to-projector | Public DTOs define what the internal projector must return. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-INPUT-DOS | Denial of Service | projection-node read | mitigate | Enforce `DEFAULT_PROJECTION_NODE_CAP + 1`, slice to cap, return cap metadata. |
| T-05-SCOPE | Elevation of Privilege | projection-node read | mitigate | Require and assert `userId` and `traceId` filters in query and params. |
| T-05-LEAK | Information Disclosure | projection DTOs | mitigate | Keep database row shapes out of API types and omit raw ClickHouse terms. |
| T-05-ANCESTRY | Tampering | graph model contract | mitigate | Assert no ancestry path fields are added; explicit edges remain source of truth. |
</threat_model>
