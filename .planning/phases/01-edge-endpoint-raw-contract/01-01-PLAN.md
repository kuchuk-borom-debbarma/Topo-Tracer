## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller-to-LogServiceImpl | Untrusted ingest objects enter the service contract before persistence. |
| LogServiceImpl-to-ILogWriteRepo | Validated ingest data crosses into persistence through a repository contract. |
| LogServiceImpl-to-IEventBus | Read-model work request is published only after persistence returns successfully. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01 | Tampering | `LogServiceImpl.validateEdgeStarts` | mitigate | Reject missing, non-string, blank, or whitespace-only `fromNodeId` and `toNodeId` before repo persistence or event publish. |
| T-02 | Denial of Service | `LogServiceImpl.validateEdgeStarts` | mitigate | Allow self-edges and unknown endpoint ids; avoid ClickHouse existence lookups in the ingest path. |
| T-01-LOG | Information Disclosure | `LogServiceImpl` logging | mitigate | Keep logs to `userId` and counts; do not log raw edge `data` or full ingest payloads. |
| T-01-OWN | Spoofing | `LogServiceImpl` user ownership propagation | mitigate | Preserve caller-provided `userId` on repository input and event data; do not invent auth/session scope in Phase 1. |
| T-01-PUB | Repudiation/Tampering | `LogServiceImpl.ingestNodesNEdges` | mitigate | Keep `await this.writeRepo.ingestNodesNEdges(data)` before `IEventBus.publish`; test failing repo does not publish. |
| T-01-SC | Tampering | npm/bun package installs | accept | No package install tasks are planned; no slopcheck checkpoint required. |
