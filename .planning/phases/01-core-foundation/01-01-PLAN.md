# Plan: Phase 1 - Core SDK Foundation & Server Ingestion

Phase 1 establishes the communication link between the Node.js SDK and the Hono backend. We will implement the ingestion endpoint on the server and the base SDK project with a fluent API and AsyncLocalStorage context.

## Phase 1.1: Hono Server Ingestion Endpoint
- [ ] **1.1.1: Implement Ingestion Route**
  - Add `POST /api/v1/ingest` in `hono-server/src/index.ts`.
  - Wire it to `logService.ingestNodesNEdges`.
  - Add basic extraction for `X-API-Key` and `userId`.
- [ ] **1.1.2: Add Route Validation**
  - Implement a simple validator to ensure `nodeStarts`, `edgeStarts`, etc. are present in the body.
- [ ] **1.1.3: Verify Ingestion Route**
  - Add a unit test or use `curl` to verify the endpoint returns `200` when given valid data.

## Phase 1.2: SDK Base Structure (Node.js)
- [ ] **1.2.1: Initialize SDK Project**
  - Create `sdks/node-js/package.json` with Bun scripts.
  - Set up `tsconfig.json`.
- [ ] **1.2.2: Define Core Types**
  - Create `sdks/node-js/src/types.ts` based on server's `IngestNodeStart`, etc.
- [ ] **1.2.3: Implement Tracer & Span (Fluent API)**
  - Create `sdks/node-js/src/Span.ts` with fluent methods (`setData`, `end`).
  - Create `sdks/node-js/src/Tracer.ts` with `startNode` and `AsyncLocalStorage` integration.
- [ ] **1.2.4: Implement HTTP Client**
  - Add basic ingestion client using `fetch` in `sdks/node-js/src/Tracer.ts`.

## Phase 1.3: Verification & Packaging
- [ ] **1.3.1: Integration Test**
  - Create `sdks/node-js/tests/integration.test.ts` to verify E2E flow (SDK -> Server).
- [ ] **1.3.2: Prepare for NPM**
  - Configure `package.json` with entry points (`main`, `module`, `types`).
  - Add a build script (e.g., using `tsup` or `bun build`).

## Verification Plan
- **Automated Tests:**
  - `bun test` in `hono-server` for the new route.
  - `bun test` in `sdks/node-js` for the SDK logic and integration.
- **Manual Check:**
  - Run a sample script using the SDK and verify data appears in ClickHouse (via logs or existing graph API).
