<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| materializer -> ILogReadRepo | Materializer will send untrusted trace/user identifiers to repository methods; repository must scope every query by both. |
| ILogReadRepo -> ClickHouse | Repository converts typed requests into SQL queries and JSONEachRow insert/query mappings. |
| raw events -> read model | Raw rows may contain malformed lifecycle timing or nullable endpoints that later materialization must diagnose. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure/Spoofing | `LogReadRepoClickHouse` load queries | mitigate | Every repository method filters by `user_id = {userId:String}` and `trace_id = {traceId:String}` and tests assert both filters. |
| T-03-02 | Tampering | checkpoint/latest-state loads | mitigate | Latest reads use grouped `argMax`/`max` selection; raw reads use explicit checkpoint tuple comparisons. |
| T-03-03 | Denial of Service | malformed raw rows | mitigate | Repository returns nullable raw row fields without throwing; Plan 02 diagnoses and continues during fold. |
| T-03-04 | Information Disclosure | repository logs | mitigate | Keep logs to userId, traceId, counts, and durations; do not log raw node/edge payloads. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs are planned; Package Legitimacy Audit says none. |
</threat_model>
