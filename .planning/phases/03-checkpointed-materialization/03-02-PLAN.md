<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw rows -> materializer | Raw node/edge rows may contain missing lifecycle fields, invalid importance, orphan endpoints, cycles, or timing anomalies. |
| materializer -> ILogReadRepo | Materializer writes replacement read state and checkpoint rows through repository contracts. |
| trace event ordering -> checkpoint | Materializer trusts trace-local event delivery order and resumes only after explicit stored bookmarks. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure/Spoofing | `TraceReadModelMaterializer.materializeTrace` | mitigate | Pass `userId` and `traceId` into every repo call; never materialize by trace id alone. |
| T-03-02 | Tampering | checkpoint advancement | mitigate | Tests assert `saveReadModel` happens before `saveCheckpoint`, and checkpoint failure does not skip raw rows on retry. |
| T-03-03 | Denial of Service | malformed graph/lifecycle data | mitigate | Diagnose and continue for missing starts/ends, invalid importance, negative duration, cycles, orphan edges, and clock skew. |
| T-03-04 | Information Disclosure | materializer logs | mitigate | Log only safe ids, counts, diagnostics, materializedAt, and durations; do not log raw node/edge payloads. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs are planned; Package Legitimacy Audit says none. |
</threat_model>
