  <files>hono-server/src/common/types.ts, hono-server/src/services/log/internal/service-impl/LogServiceImpl.ts</files>
  <action>
    1. Update `hono-server/src/common/types.ts`: Define `ConflictError` extending `TopoTraceException` with `statusCode: 409`.
    2. Update `LogServiceImpl.projectTraceGraph` to handle `cursor` and `limit` (per D-14, D-15):
       a. Change signature: `projectTraceGraph(data: { userId: string; traceId: string; threshold: number; cursor?: string; limit?: number; })`.
       b. Enforce `limit`: Use provided `limit` or `DEFAULT_PROJECTION_NODE_CAP` (500), capped at 1000.
       c. Fetch `summary` via `readRepo.loadTraceSummary`. If null, throw "Trace not found".
       d. Decode `cursor`:
          - If no cursor: `offset = 0`, `expectedAt = summary.materializedAt`.
          - If cursor: `decodeCursor(cursor)`. If `materializedAt !== summary.materializedAt`, throw a `ConflictError` (per D-03).
       e. Fetch nodes: Use `readRepo.loadBoundedProjectionNodes` with calculated `offset` and `limit`.
       f. Fetch edges: Use `readRepo.loadBoundedVisibleEdges` for the fetched nodes.
       g. Assemble `metadata.paging` (per D-16, D-17, D-18, D-19):
          - `hasAfter = boundedNodes.hasMore`
          - `hasBefore = offset > 0`
          - `nextCursor`: `encodeCursor(offset + limit, summary.materializedAt)` if `hasAfter`.
          - `previousCursor`: `encodeCursor(max(0, offset - limit), summary.materializedAt)` if `hasBefore`.
          - `totalNodeCount = summary.nodeCount`.
          - `fromFlowOrder / toFlowOrder`: Flow orders of first/last items in `boundedNodes.items`, or 0 if empty.
  </action>
  <verify>
