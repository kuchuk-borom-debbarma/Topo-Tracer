# 1.2. API Contracts

The Hono backend exposes JSON endpoints under `/api/v1`. Authenticated trace routes accept either:

- `Authorization: Bearer <jwt>` for browser/session usage;
- `X-API-Key: <api-key>` for SDK ingestion.

`jwtAuthMiddleware` resolves the user, sets `userId` in the Hono context, and rejects `X-User-Id` tenant mismatches when that header is supplied.

## Authentication

### `POST /api/v1/auth/signup/start`

Starts OTP-backed signup.

```json
{
  "username": "ada",
  "email": "ada@example.com",
  "password": "secret"
}
```

Response:

```json
{
  "token": "token-otp-row-id"
}
```

### `POST /api/v1/auth/signup/finish`

```json
{
  "token": "token-otp-row-id",
  "otp": "123456"
}
```

Response:

```json
{
  "success": true
}
```

### `POST /api/v1/auth/login`

```json
{
  "email": "ada@example.com",
  "password": "secret"
}
```

Response:

```json
{
  "token": "jwt"
}
```

### `GET /api/v1/auth/me`

Returns the authenticated user:

```json
{
  "user": {
    "id": "user-id",
    "username": "ada",
    "email": "ada@example.com"
  }
}
```

### `GET /api/v1/auth/api-keys`

Returns API keys owned by the authenticated user.

### `POST /api/v1/auth/api-keys`

```json
{
  "name": "local-sdk"
}
```

Response contains the plaintext key once:

```json
{
  "apiKey": {
    "id": "key-id",
    "name": "local-sdk",
    "keyPrefix": "tt_abc123",
    "key": "tt_full_secret_key",
    "createdAt": "2026-06-19T00:00:00.000Z",
    "lastUsedAt": null,
    "revokedAt": null
  }
}
```

### `DELETE /api/v1/auth/api-keys/:apiKeyId`

Revokes an API key for the authenticated user.

### Password reset

`POST /api/v1/auth/reset-password/start` accepts `{ "email": "..." }` and returns a token. `POST /api/v1/auth/reset-password/finish` accepts `{ "token": "...", "otp": "...", "newPassword": "..." }`.

## Telemetry Ingestion

### `POST /api/v1/ingest`

Requires a JWT or API key.

The route requires `nodeStarts`, `edgeStarts`, `nodeEnds`, and `edgeEnds` keys. `traceStarts` is optional and defaults to an empty array in service code.

```json
{
  "traceStarts": [
    {
      "traceId": "trace-uuid",
      "name": "Checkout Flow Demo",
      "importanceLabels": {
        "0": "request",
        "1": "work",
        "2": "detail"
      },
      "timestamp": 1781480976387
    }
  ],
  "nodeStarts": [
    {
      "id": "node-uuid",
      "traceId": "trace-uuid",
      "nodeType": "db-call",
      "data": {
        "query": "select-cart"
      },
      "name": "CartRepo.loadCart",
      "startMessage": "load-cart-db",
      "startedAt": 1781480976388,
      "importanceLevel": 1
    }
  ],
  "edgeStarts": [
    {
      "id": "edge-uuid",
      "traceId": "trace-uuid",
      "edgeType": "child",
      "fromNodeId": "parent-node-uuid",
      "toNodeId": "node-uuid",
      "data": {},
      "startedAt": 1781480976388
    }
  ],
  "nodeEnds": [
    {
      "id": "node-uuid",
      "traceId": "trace-uuid",
      "endedAt": 1781480976418,
      "endMessage": "done"
    }
  ],
  "edgeEnds": [
    {
      "id": "edge-uuid",
      "traceId": "trace-uuid",
      "endedAt": 1781480976418
    }
  ]
}
```

Response:

```json
{
  "success": true
}
```

Backend behavior:

- the route resolves `userId` from auth;
- `LogServiceImpl` validates edge start endpoints and publishes `log.telemetry.received`;
- `LogIngestConsumer` appends rows to ClickHouse;
- materialization happens asynchronously after `log.trace.ingested`.

## Trace Reads

### `GET /api/v1/traces?page=1&limit=20`

Lists latest materialized trace summaries for the authenticated user.

Validation:

- `page` must be a positive integer;
- `limit` must be an integer from 1 to 100.

Response:

```json
{
  "traces": [
    {
      "userId": "user-id",
      "traceId": "trace-uuid",
      "name": "Checkout Flow Demo",
      "importanceLabels": {
        "0": "request",
        "1": "work"
      },
      "nodeCount": 42,
      "edgeCount": 41,
      "minImportanceLevel": 0,
      "maxImportanceLevel": 3,
      "startedAt": 1781480976387,
      "endedAt": 1781480977000,
      "materializedAt": 1781480977010,
      "diagMissingStarts": 0,
      "diagMissingEnds": 0,
      "diagNegativeDurations": 0,
      "diagCycles": 0,
      "diagOrphanEdges": 0,
      "diagInvalidImportance": 0,
      "diagClockSkew": 0,

  ],
  "totalCount": 1,

  "limit": 20,

}
```

### `GET /api/v1/traces/:traceId/summary`

Returns one latest summary or `404` when the trace is not visible for the user.

- `threshold` must be a non-negative integer;

### `DELETE /api/v1/traces/:traceId`

Schedules asynchronous ClickHouse mutations across raw and read-model tables.

Response:

```json
{
  "accepted": true
}
```
