# API Contracts

The `hono-server` exposes a RESTful JSON API. All authenticated routes require a JWT token passed in the `Authorization: Bearer <token>` header, generated via the Auth endpoints.

## 1. Authentication (`/api/v1/auth`)

-   **`POST /api/v1/auth/signup/start`**
    -   Payload: `{ "username": "...", "email": "...", "password": "..." }`
    -   Response: `{ "token": "<jwt-signup-token>" }`
-   **`POST /api/v1/auth/signup/finish`**
    -   Payload: `{ "token": "...", "otp": "..." }`
    -   Response: `{ "success": true }`
-   **`POST /api/v1/auth/login`**
    -   Payload: `{ "email": "...", "password": "..." }`
    -   Response: `{ "token": "<jwt-auth-token>" }`
-   **`GET /api/v1/auth/me`** (Auth Required)
    -   Response: User profile data.
-   **`GET /api/v1/auth/api-keys`** (Auth Required)
    -   Response: `{ "apiKeys": [...] }`
-   **`POST /api/v1/auth/api-keys`** (Auth Required)
    -   Payload: `{ "name": "production-key" }`
    -   Response: `{ "apiKey": { ... } }`

## 2. Trace Ingestion (`/api/v1/ingest`)

**`POST /api/v1/ingest`**
*(Auth Required - can use Bearer JWT or API Key)*

Accepts raw Node and Edge lifecycle events from the SDK.

**Payload:**
```json
{
  "traceStarts": [...],
  "nodeStarts": [
    {
      "eventId": "evt-123",
      "traceId": "trace-1",
      "entityId": "node-A",
      "entityType": "node",
      "eventType": "node.started",
      "occurredAtUnixMs": 1700000000,
      "name": "DB Query",
      "importanceLevel": 1,
      "data": "{}"
    }
  ],
  "nodeEnds": [...],
  "edgeStarts": [...],
  "edgeEnds": [...]
}
```

## 3. Trace Reads (`/api/v1/traces`)

**`GET /api/v1/traces`** (Auth Required)
Lists traces belonging to the authenticated user.
-   Query Params: `page` (default: 1), `limit` (default: 20)

**`GET /api/v1/traces/:traceId/summary`** (Auth Required)
Retrieves high-level counts and status for a trace without loading the graph.

**`GET /api/v1/traces/:traceId/flow`** (Auth Required)
Loads the projected, importance-filtered graph window for the UI canvas.
-   Query Params: 
    -   `threshold`: The maximum importance level to return (e.g., `0` for critical only, `3` for detailed).
    -   `limit`: Graph node hard cap (default: 1000).
    -   `cursor`: Pagination cursor.
-   Response: `{ "nodes": [...], "edges": [...], "nextCursor": "..." }`
