# 5. Development And Verification

This guide outlines the local development setup, verification commands, and environment variables for the Topo-Tracer project.

Local defaults:
- **ClickHouse:** `http://localhost:8123`
- **hono-server (Backend):** `http://localhost:8787` (Wrangler dev default)
- **Frontend:** `http://localhost:5173`
- **Frontend API default:** `http://localhost:8787` (or `http://localhost:3999` if overridden)

---

## 1. Local Development Setup

### Start ClickHouse Locally
Ensure ClickHouse is running locally. By default, it is expected at `http://localhost:8123` with username `default` and password `password`.

### Start the Backend (hono-server)
The active backend is `hono-server`. It runs as a Cloudflare Worker using Wrangler.

```sh
cd hono-server
bun run dev
```

Expected output:
```text
Ready on http://localhost:8787
```

### Start the Frontend
The frontend is built using React and served by Vite.

```sh
cd frontend
npm run dev
```

By default, Vite runs the frontend on `http://localhost:5173`. To point the frontend to the backend Wrangler dev port:
```sh
VITE_API_BASE_URL=http://localhost:8787 npm run dev
```

## 2. Testing and Verification

### Run Unit Tests
Unit tests in `hono-server` are run via Bun's native test runner.

```sh
cd hono-server
bun test
```

This runs the entire 216-test suite covering database schemas, repositories, event-bus, outbox relay, auth services, and log projection.

### Run Codebase Quality Audits (Fallow)
Fallow checks for dead code, duplication, complexity, and architectural boundary violations.

```sh
cd hono-server
bun run fallow
```

To run a full-repo health and maintainability scan:
```sh
bun x fallow health
```

### Run TypeScript Compilation Check
Verify type safety across the backend, SDKs, and frontend.

```sh
# hono-server
cd hono-server
bun x tsc --noEmit

# Frontend
cd frontend
npm run build # runs tsc -b && vite build

# Node.js SDK
cd sdks/node-js
npm run build
```

## 3. Smoke Checking the APIs

### List Materialized Traces
```sh
curl -s -H "Authorization: Bearer <JWT_TOKEN>" \
  'http://localhost:8787/api/v1/traces?limit=3'
```

### Fetch Bounded Trace Flow Projection
```sh
curl -s -H "Authorization: Bearer <JWT_TOKEN>" \
  'http://localhost:8787/api/v1/traces/{traceId}/flow?threshold=2&limit=100'
```

## 4. Environment Variables

### hono-server Bindings (via Wrangler or Env)
- `CLICKHOUSE_URL`: ClickHouse connection URL, default `http://localhost:8123`.
- `CLICKHOUSE_USERNAME`: ClickHouse username, default `default`.
- `CLICKHOUSE_PASSWORD`: ClickHouse password, default `password`.
- `CLICKHOUSE_DATABASE`: ClickHouse database name, default `default`.
- `JWT_SECRET`: Secret key used for signing and verifying JWT tokens.
- `DISABLE_SELF_TRACING`: Set to `true` to disable backend self-tracing spans.

### Frontend
- `VITE_API_BASE_URL`: The URL of the API gateway (pointing to `hono-server`), default `http://localhost:8787`.
