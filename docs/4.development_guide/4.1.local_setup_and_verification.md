# 4.1. Local Setup and Verification

This guide covers local backend development for Topo-Tracer.

## Services

Expected local defaults:

```txt
ClickHouse  http://localhost:8123
Postgres    postgres://postgres:password@localhost:5432/topo_tracer
Backend     http://localhost:8787 through Wrangler
Frontend    http://localhost:5173 through Vite
```

## Backend

```sh
cd hono-server
bun install
bun run dev
```

`bun run dev` runs `wrangler dev`. Wrangler uses `hono-server/wrangler.jsonc`, whose entry point is `src/index.ts`.

The package also contains `src/bun.ts`, which serves the same Hono app on `PORT` or `3999`. There is no package script for it today.

## Frontend

```sh
cd frontend
npm install
VITE_API_BASE_URL=http://localhost:8787 npm run dev
```

If `VITE_API_BASE_URL` is omitted, the frontend uses an empty base URL and expects the API to be available on the same origin.

## Node SDK Demos

Create an API key in the UI, then run:

```sh
cd sdks/node-js
bun install
TOPO_TRACER_URL=http://localhost:8787 bun run demo:e2e
```

The demo prompts for the API key and sends it as `X-API-Key`.

Useful SDK scripts:

```sh
bun run demo:e2e
bun run demo:stress
bun run build
bun test
```

## Backend Verification

Run from `hono-server`:

```sh
bun test
bun run fallow
bun x tsc --noEmit
```

What this covers:

- route validation and auth checks;
- Postgres and ClickHouse schema helpers;
- event bus idempotency;
- outbox claiming, recovery, and relay behavior;
- auth service and middleware;
- log service orchestration;
- raw/read repository mapping;
- materialization edge cases;
- flow ordering;
- projection and cursor behavior.

## Frontend Verification

```sh
cd frontend
npm run build
```

The frontend build runs `tsc -b && vite build`.

## Smoke Checks

Login:

```sh
curl -s http://localhost:8787/api/v1/auth/login \
  -H 'content-type: application/json' \
  -d '{"email":"ada@example.com","password":"secret"}'
```

List traces:

```sh
curl -s 'http://localhost:8787/api/v1/traces?page=1&limit=10' \
  -H "Authorization: Bearer <JWT_TOKEN>"
```

Fetch a graph window:

```sh
curl -s 'http://localhost:8787/api/v1/traces/<traceId>/flow?threshold=2&limit=160' \
  -H "Authorization: Bearer <JWT_TOKEN>"
```

Ingest with an API key:

```sh
curl -s http://localhost:8787/api/v1/ingest \
  -H 'content-type: application/json' \
  -H "X-API-Key: <API_KEY>" \
  -d '{
    "traceStarts": [],
    "nodeStarts": [],
    "edgeStarts": [],
    "nodeEnds": [],
    "edgeEnds": []
  }'
```

An empty batch returns success from the service path, but it creates no raw rows.
