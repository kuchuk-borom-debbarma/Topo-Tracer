# 6.1. Hono Codebase Guide

This guide is for backend engineers changing `hono-server/src`.

## Layering

```txt
src/
  common/          shared logger, env, error types
  infra/           database clients, auth middleware, event bus, tracing, cache
  services/
    auth/          auth public API and internals
    log/           trace ingestion, materialization, projection
    external-notification/
  index.ts         Hono app and HTTP routes
  bun.ts           optional Bun server wrapper
```

Routes adapt HTTP to services. Services own workflows. Repositories own persistence. Infrastructure owns platform capabilities.

## Route Rules

Routes in `src/index.ts` should:

- validate request shape and query parameters;
- read auth context from middleware;
- call one service method;
- translate expected errors to HTTP responses;
- stay free of SQL, ClickHouse queries, and long business workflows.

Trace routes currently mounted:

- `POST /api/v1/ingest`
- `GET /api/v1/traces`
- `GET /api/v1/traces/:traceId/summary`
- `GET /api/v1/traces/:traceId/flow`
- `DELETE /api/v1/traces/:traceId`

## Log Service Module

```txt
services/log/
  api/
    ILogService.ts
    types.ts
  internal/
    materialization/
    projection/
    repo/
    service-impl/
    util/
    worker/
  index.ts
```

Public contract:

- `ingestNodesNEdges`
- `projectTraceFlow`
- `listTraces`
- `getTraceSummary`
- `deleteTrace`

Use `api/types.ts` for public shapes consumed across layers. Keep ClickHouse row shapes in `internal/repo/types.ts`.

## Ingestion Path

The route does not write raw rows. It calls `LogServiceImpl.ingestNodesNEdges`, which publishes `log.telemetry.received`.

`LogIngestConsumer` owns raw ClickHouse appends and then emits `log.trace.ingested`.

Change ingestion validation in `LogServiceImpl`. Change raw row mapping in `LogWriteRepoClickHouse`.

## Materialization Path

`ReadOptimisedAggregator` owns rebuild scheduling:

- dedupe repeated trace ids in one batch;
- serialize rebuilds for one `{userId, traceId}`;
- cap concurrent rebuilds with `p-limit`.

`TraceReadModelMaterializer` owns trace reconstruction:

- load checkpoint;
- load latest read model;
- replay new raw events;
- compute flow order;
- correct clock skew;
- build summary diagnostics;
- save read model;
- save checkpoint.

Do not move this logic into routes or repositories.

## Projection Path

`LogServiceImpl.projectTraceFlow` owns request-level rules:

- load summary;
- reject self-traces;
- validate cursor freshness;
- load bounded projection nodes and edges;
- call `LogFlowProjector`;
- fill paging metadata.

`LogFlowProjector` owns pure projection:

- sort by `flowOrder`;
- apply `importanceLevel <= threshold`;
- collapse hidden runs into ghost nodes;
- snap and aggregate edges.

## Storage Rules

ClickHouse:

- raw events use `MergeTree`;
- read models use `ReplacingMergeTree`;
- latest reads should use `argMax(..., materialized_at_ms)`;
- checkpoints are the source of materialization progress;
- deletion uses asynchronous mutations.

Postgres:

- auth and outbox tables live in `infra/db/postgres/schema.ts`;
- auth repositories should keep SQL inside repo implementations;
- transactional event publishing should use the outbox path.

## Event Bus Rules

Use `IEventBus`, not concrete bus implementations, from service code.

Event names currently use dotted topic names:

- `log.telemetry.received`
- `log.trace.ingested`
- `auth.signup.started`

Provide:

- stable `idempotencyId`;
- useful `key`;
- small `data`;
- optional `batchId` for publish correlation.

Consumers must be idempotent.

## Auth and Tenant Boundaries

`jwtAuthMiddleware` supports:

- Bearer JWTs;
- `X-API-Key`;
- optional `X-User-Id` tenant mismatch detection.

Service calls that operate on user-owned data should accept `userId` explicitly. Do not let trace reads or deletes cross tenant boundaries.

## Runtime Environment

Use `common/env.ts` helpers where Hono context is available. Some boot-time code currently reads `process.env` directly because background daemons start before request context exists; keep that boundary explicit.

## Verification

Run from `hono-server`:

```sh
bun test
bun run fallow
bun x tsc --noEmit
```

Use focused tests near the changed layer:

- event bus and outbox: `src/infra/event-bus/**/*.test.ts`;
- auth: `src/services/auth/**/*.test.ts`;
- materializer: `src/services/log/internal/materialization/**/*.test.ts`;
- projection: `src/services/log/internal/projection/**/*.test.ts`;
- repositories: `src/services/log/internal/repo/**/*.test.ts`;
- routes: `src/index.flow.test.ts`.
