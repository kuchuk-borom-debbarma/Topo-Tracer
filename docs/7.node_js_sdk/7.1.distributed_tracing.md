# 7.1. Node.js SDK Tracing Model

The Node.js SDK in `sdks/node-js/src` emits the same lifecycle event shapes accepted by `hono-server`.

## Core API

```ts
import { Tracer, NodeType, Importance } from "@topo-tracer/node-sdk";

const tracer = new Tracer({
  endpoint: "http://localhost:8787",
  apiKey: "tt_secret",
  serviceName: "checkout-api",
  batchSize: 50,
  flushInterval: 5000,
});

await tracer.trace(
  "checkout-request",
  async (span) => {
    span.setAttribute("customer.id", "cust_demo_001");

    await tracer.trace(
      "load-cart-db",
      async (child) => {
        child.setAttribute("storage.kind", "cache");
        tracer.log("Cart found in cache", { cartId: "cart_123" });
      },
      { type: NodeType.DB_CALL },
    );
  },
  {
    type: NodeType.CONTROLLER,
    traceName: "Checkout Flow Demo",
    importanceLabels: {
      0: "request",
      1: "work",
      2: "detail",
    },
  },
);

await tracer.flush();
```

## What The SDK Emits

The SDK buffers:

- `traceStarts` for root trace metadata;
- `nodeStarts` when a span begins;
- `nodeEnds` when a span ends;
- `edgeStarts` for parent-child links;
- `edgeEnds` when explicit edge ending is added by callers or integrations.

Nested `tracer.trace` calls are linked using `AsyncLocalStorage`. The SDK creates explicit `edgeStarts` from parent span id to child span id.

## Grouping and Layers

Spans can also carry visual grouping metadata:

```ts
await tracer.trace("controller", async () => {
  await tracer.trace("foo", async () => {});

  await tracer.trace("payments-api", async () => {}, {
    type: NodeType.REMOTE_CALL,
    groupParentId: null,
    layer: {
      key: "external-services",
      label: "External Services",
      order: 3,
    },
  });
});
```

Defaults:

- nested spans use the active span id as `groupParentId`;
- root spans use `groupParentId: null`;
- `layer` is omitted unless provided.

Use `groupParentId: null` plus a shared `layer` to render service-style peers outside the current function group while preserving the normal edge from caller to callee. The SDK does not expose a special `service()` method.

## Logs As Nodes

`tracer.log()` creates a short-lived node with `nodeType: "log"`. This lets logs appear in the graph flow instead of being buried as text inside another span.

```ts
tracer.log("Payment gateway responded", {
  status: "success",
  latency: "45ms",
}, Importance.HIGH);
```

## Importance

The backend displays nodes when:

```txt
importanceLevel <= selectedThreshold
```

The SDK supports explicit importance through `Importance` enum values or numeric values. If no explicit importance is provided, it maps common node types such as controllers, HTTP calls, DB calls, IO, and logs to defaults, and otherwise increases importance relative to the active parent span.

## Context Helpers

The current SDK exposes:

- `extractContext()`: returns the active `{ traceId, spanId }` from AsyncLocalStorage.
- `injectContext({ traceId, spanId })`: returns a synthetic external `Span` for a supplied context.

It does not currently mutate HTTP header objects directly. If an integration needs wire propagation, build it around these returned ids or add a focused helper in the SDK.

## Sending Data

The SDK posts to `/api/v1/ingest`. If `endpoint` already ends with `/api/v1/ingest`, it uses that exact URL; otherwise it appends `/api/v1/ingest`.

Headers:

```txt
Content-Type: application/json
X-API-Key: <apiKey>
X-User-Id: <userId> when configured
```

The backend normally derives the user from the API key. `X-User-Id` is optional and is checked by middleware when present.
