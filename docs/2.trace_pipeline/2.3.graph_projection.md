# Graph Projection

While the `TraceReadModelMaterializer` computes the absolute reality of every node and edge in a trace, the UI rarely wants to render a 10,000-node graph simultaneously. 

The `LogFlowProjector` sits between the materializer and the HTTP response, actively shrinking the graph down based on the user's selected **importance threshold** and **pagination limit**.

## 1. Threshold Filtering
Nodes are assigned a semantic importance level (e.g., `0` for CRITICAL, `3` for DETAIL). The projector first discards any nodes whose `importanceLevel` is greater than the user's `threshold`.

## 2. Ghost Nodes
When nodes are filtered out, the graph could visually break apart. To preserve the structure, the projector collapses contiguous chunks of hidden nodes into single `ghost:hidden:N` nodes.

For example, if the trace goes:
`A (Visible) -> B (Hidden) -> C (Hidden) -> D (Visible)`

The projector outputs:
`A (Visible) -> Ghost -> D (Visible)`

This keeps the canvas readable while indicating that work was skipped.

## 3. Edge Lifting
Edges that pointed to hidden nodes are "lifted" to point to the corresponding ghost node instead. If an edge connects two hidden nodes within the same ghost segment, it is discarded from the UI entirely, as it does not cross any visible boundaries.

## 4. Windowing
Finally, the projector slices the visible and ghost nodes by `flowOrder` using the `cursor` and `limit` parameters, returning a `GraphWindowResponse` containing only the subset of the graph the user is currently looking at.
