<task type="auto">
  <name>Task 1: Create verification script verify-trace-names.ts</name>
  <files>../verify-trace-names.ts</files>
  <action>
    Create a script named `verify-trace-names.ts` in the project root that performs the following steps:
    1. Import `Tracer` from the built SDK (`./sdks/node-js/dist/index.js`).
    2. Initialize the `Tracer` with:
       - `endpoint: "http://localhost:3333/api/v1/ingest"`
       - `apiKey: "dev-api-key"`
       - `userId: "dev-user-id"`
    3. Start a trace with `traceName: "E2E Verification Trace"`.
    4. Start a child node within that trace.
    5. End nodes and `flush()` the tracer.
    6. Wait for a few seconds (e.g., 2s) for materialization to complete in the backend.
    7. Fetch all traces from `http://localhost:3333/api/v1/traces` using `fetch`.
    8. Verify:
       - The recently created trace exists and its `name` matches "E2E Verification Trace".
       - Create another trace *without* a name and verify it falls back correctly (e.g., name is undefined or defaults to ID in summary).
    9. Exit with code 1 if any verification fails, or code 0 if all pass.

    Ensure the script includes clear console logging for each step and failure reason.
  </action>
  <verify>
    <automated>test -f ../verify-trace-names.ts</automated>
  </verify>
  <done>The verification script is created with the required logic.</done>
</task>
